## Key Conventions

### Source vs. Generated Files
- `.md` files are the source of truth for `index.html`, `portfolio/index.html`, `credentials/index.html`, `contact/index.html`, and `404.html`
- `portfolio/index.md` and `credentials/index.md` are themselves **generated** from JSON — edit the JSON, not the markdown
- Never hand-edit any `.html` file; always rebuild via `make`

### Markdown Frontmatter
Every `.md` source file begins with a YAML-like block between `---` delimiters:

```markdown
---
title: ((en))Page Title((/en))((es))Título((/es))((fr))Titre((/fr))((pt))Título((/pt))
lang: en
base_href: ./
nav_current: 1          # 1=home, 2=portfolio, 3=credentials, 4=contact
js: [cookies, language, theme, menu, favicon]
css: [theme, common]
---
```

### i18n Syntax
Multilingual strings use inline tags — always provide all supported languages (en, es, fr, pt):

```
((en))English text((/en))((es))Texto en español((/es))((fr))Texte en français((/fr))((pt))Texto em português((/pt))
```

The generator wraps paired blocks in `<span class="i18n" data-i18n-en="..." data-i18n-es="..." data-i18n-fr="..." data-i18n-pt="...">`.
`language.js` swaps content at runtime based on cookie or browser preference.

### CSS Architecture
- All color values must use CSS custom properties defined in `theme.css` — no hardcoded colors
- `theme.css` defines the Nord palette and both dark (default) and light (`.theme-light`) themes
- Add page-specific styles in a new `css/<page>.css` and reference it via frontmatter

### JavaScript Architecture
- All JS files are global-scope modules (no ES module `import/export` at runtime)
- `cookies.js` must be loaded before `language.js` and `theme.js` (declared order in frontmatter matters)
- Functions are documented with JSDoc `@param` / `@returns` comments
- Use double quotes for strings (ESLint rule enforced)
- `/* eslint-disable no-unused-vars */` is used at the top of files that expose globals

### Data Files
- `portfolio/data/projects.json` — edit to add/update projects, then run `make portfolio`
- `credentials/data/credentials.json` — edit to add/update credentials, then run `make credentials`
- Both JSON files have accompanying `.schema.json` files defining their structure
